# M16 — School Administration & Configuration — ✅ STEPS 1–9 COMPLETE (awaiting approval)

Scope: a **school administration panel** over frozen M1–M15 — configure branding, school profile, numbering, academic
defaults, language, timezone, theme and the working week. **Completely additive: three new config tables, one new private
bucket, one new permission; zero frozen-table ALTER, no business-logic change.** **Configuration influences only future
actions and never rewrites a historical record.** **ADR-024** governs the design.

> Everything is **configuration**. No attendance/exam/fee/report-card/timetable logic changes. Several stored settings
> (numbering, timezone, language, academic defaults) are **read by no frozen engine in v1** — the ADR-014/ADR-023
> "seam-provisioned, wiring-deferred" posture; wiring a frozen domain to consume config is a future per-domain change.

## Permissions

**One added** — `settings:manage` (SA/OA). Writes only; reads are a role-shaped service projection (no read grant).
**No feature flag.** Teacher/Parent/Accountant: none.

## Deviations / v1 simplifications (all flagged + approved)

1. **Configuration is inert w.r.t. every frozen engine** — numbering/timezone/language/academic settings stored but read
   by no engine in v1 (ADR-024 §5); wiring is a future per-domain change.
2. **`AcademicSettings` folded into `SchoolSettings`** — same admin audience; a JSON escape-hatch for compound defaults.
3. **One permission, not a permission-configuration system** — "Office Admin: configured permissions" → a fixed
   `settings:manage` grant.
4. **Audit-history viewer + audit shortcut deferred** — no audit-read surface exists; changes are still audited.
5. **"Backup/export settings" = the CSV export** of the current configuration.
6. **Mobile admin-editing limited to theme + language** — full profile editing on the web console (the M13/M15 precedent).

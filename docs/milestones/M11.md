## Permissions

`announcement:read` (SA/OA/T/P) В· `announcement:manage` (SA/OA вЂ” full lifecycle incl. publish/archive) В·
`announcement:draft` (TEACHER вЂ” own SECTION/CLASS drafts, no publish) В· `calendar:read` (SA/OA/T/P). Calendar
**writes reuse `academic:manage`** (no new write permission). **Permission-only вЂ” no feature flag.**

## Deviations (ADR-019)

1. M10 stays frozen; M11 is a new persistent domain that *optionally* emits M10 notifications.
2. No `CUSTOM` audience (dropped at Step-1 approval; future-additive with a recipient table).
3. "Published visible if targeted" is business + coarse RLS, not RLS alone (no recipient table в†’ per-user RLS
   targeting isn't cheap; the app is `service_role`/BYPASSRLS).
4. Teachers draft, admins publish (Step-1 approval).
5. Calendar `EXAM` events are manual, not synced from frozen M5.
6. One disclosed touch of the M10 mobile inbox (`open()` prefers `actionUrl`) for announcement deep-linking.

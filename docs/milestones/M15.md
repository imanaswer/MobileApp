# M15 — Documents, Certificates & Downloads — ✅ STEPS 1–9 COMPLETE (awaiting approval)

Scope: a **secure per-student document center** over frozen M1–M14 — the school **issues** certificates (generated or
uploaded), runs an **approval workflow**, and parents **download** approved documents via 60s signed URLs. **Purely
additive: two new tables, one new private bucket, three new permissions; zero frozen-table ALTER, no business-logic
change.** **ADR-023** governs the design.

> `Document` is **distinct from M3 `StudentDocument`** (KYC/identity uploads with type-visibility) — it's a school-ISSUED
> record with a GENERATED/UPLOADED→APPROVED→ARCHIVED lifecycle. Generation is **metadata-first**: values are frozen into
> `snapshotJson` at issue (the ADR-014 report-card snapshot philosophy), so a later profile change can't rewrite an issued
> certificate. No PDF/HTML renderer exists yet (the ReportCard `pdfPath`-deferred precedent); the **upload** path is the
> fully-working v1 route, rendering is the named deferred upgrade.

## Permissions

**Three added** (`packages/constants/permissions.ts`): `document:manage` + `document:approve` (SA/OA) and `document:read`
(SA/OA all · TEACHER own-section · PARENT own-child). Non-admins see **APPROVED only** (the report-card PUBLISHED-only
precedent). **No feature flag.** Accountant: none.

## Deviations / v1 simplifications (all flagged + approved)

1. **Generation renders no file** — metadata + `snapshotJson` only (`storagePath` null, `hasFile` false); the ReportCard
   `pdfPath`-deferred precedent. HTML/PDF rendering is the named upgrade behind the same service method; the **upload path
   is fully working**.
2. **`REPORT_CARD` / `FEE_RECEIPT` are UPLOADED slots** — no cross-domain render (report-card PDF stays ADR-014's seam).
3. **Teacher scope narrowed to own-section** (`teaches_student`) — the M3 `StudentDocument` precedent; tighter than the
   original ADR §6 "school" draft.
4. **Upload is web-only** on the client — no mobile file picker (the homework/announcement precedent).
5. **A deleted draft's bucket object is orphaned** — `StoragePort` has no delete; named ceiling (`StoragePort.deleteObject`
   is the deferred upgrade).
6. **New school-wide `listSchoolDocuments`** added for the admin console/approval queue (the "Step 7 pulls it in" case).

## Out of scope
